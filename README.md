## Overview

<a href="#">Warp for Mac</a> extends its core terminal with Warp Drive team collaboration, Notebook mode for documented workflows, agent mode for autonomous task execution, and deep shell integration that make it a complete developer productivity platform rather than a terminal emulator with AI features bolted on. These capabilities address the full range of how developers use the terminal — individual productivity, team knowledge sharing, and complex automated workflows.

<a href="#">Warp Drive</a> is a team workspace within Warp where command workflows, runbooks, and documented shell scripts live as shared resources that all team members access directly from within their terminal sessions. A database migration runbook, a deployment checklist, a debugging workflow for a common production issue — all stored in Warp Drive and available with a search from any team member's terminal without navigating to documentation sites, Notion pages, or Confluence. <a href="#">Workflow sharing</a> enables team members to save a sequence of commands with documentation and make them available to the entire team, turning individual command expertise into shared organizational knowledge that onboards new team members faster and standardizes how common tasks are performed.

<a href="#">Agent mode</a> executes multi-step terminal tasks autonomously from a natural language goal description — a developer describes what needs to happen and Warp AI generates and executes the necessary commands, checks the output, handles errors, and proceeds through the steps without requiring manual command-by-command supervision. Installing a dependency, configuring a development environment, running a series of data processing steps, or setting up a new service all become descriptions of intent rather than sequences of commands. <a href="#">Human-in-the-loop approval</a> presents each planned command before execution for review and approval, giving developers control over what agent mode executes without requiring them to write each command manually.

<a href="#">SSH session management</a> handles remote server connections with the same block-based interface and AI assistance available for local terminal sessions — remote commands produce navigable blocks, AI generates commands for the remote environment, and session state persists across connection drops and reconnections. <a href="#">Panes and tabs</a> organize multiple simultaneous terminal sessions in split-pane layouts and tabbed windows, enabling side-by-side local and remote sessions, multiple project directories open simultaneously, and organized multi-session workflows that complex development environments require.

## Key Features

- <a href="#">Warp Drive team workflow library</a> stores shared command workflows runbooks and scripts accessible from within any team member's terminal session
- <a href="#">Workflow knowledge sharing</a> converts individual command expertise into searchable team resources for faster onboarding and task standardization
- <a href="#">Agent mode autonomous execution</a> generates and runs multi-step terminal tasks from natural language goal descriptions with output monitoring
- <a href="#">Human-in-the-loop approval</a> presents each planned command for review before agent mode execution for developer oversight and control
- <a href="#">SSH remote session management</a> applies block-based interface and AI assistance to remote server connections with persistent session state
- <a href="#">Split-pane and tabbed layout</a> organizes multiple simultaneous sessions in configurable pane arrangements and named tabs
- <a href="#">Custom themes and appearance</a> configures terminal colors fonts cursor styles and background images through a comprehensive theme system
- <a href="#">Shell integration hooks</a> connects deeply with zsh bash and fish for accurate prompt detection command timing and environment awareness
- <a href="#">Command history fuzzy search</a> finds any previous command through full-text fuzzy search across the complete session and persistent history

## Additional Information

Warp Drive addresses the knowledge management problem that terminal-heavy development workflows create — the expertise embedded in individual developers' command histories, shell aliases, and personal runbooks is invisible to the rest of the team and lost when those developers leave. By making team workflows a first-class feature of the terminal itself rather than external documentation, Warp creates a living repository of operational knowledge that improves with each team member's contribution.

Agent mode represents the most ambitious extension of AI assistance in the terminal — moving from AI that suggests commands to AI that executes goals. The human-in-the-loop design makes this automation safe by preserving developer oversight at each step, combining the speed of AI-generated command sequences with the judgment of a human reviewer who understands the intent and can catch errors before they affect the system.
